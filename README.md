# 概要

小小輸入法では `x` を特定の時間押しっぱなしにしたまま`y` を押したときを検知して `xy+` の綴で表せる。また、設定文件に `code_rule` を記述して熟語を登録するときにその字数と各文字の綴をもとに既定の綴を決められる。
（後述の`code_rule`章を参照）
これを利用して以下を実現する。
- 漢字一文字の入力と複数文字の熟語を綴の接頭辞で区別
- 拼音を綴に組み込む

そこで、単漢字綴は2,3文字目の後ろに`+`（同時打鍵できないときは `.` または `,`）を追加することにする。

無作為な試行からなるべく同時打鍵できる綴の多い入れ替え方を探す。


## 注意
`sdfjkl`列が全て文字になるようqwerty配列の下段を変更した以下の鍵配列を想定している。
```
qwert  yuiop
asdfg  hjkl
 zxc.b ,mnv
```
- 運指を左右対称にしたい
- `+` は記号類の後ろでは同時打鍵として扱えない（文字通りの`+`の入力しか受け付けない）
- 他の綴との兼ね合い

という理由による。

さもなくば以下で対応できるかも
- `spell/fluent.txt` に同時打鍵できる鍵列を書く
- 同時打鍵できないとき、3文字目が左手側なら `,`、右手側なら `.` を後ろに付け足す挙動を変更。

# 使い方
## `cargo run l level <i> <j> [<source>] - [<keys> <chars>]`

`l` は loop から。無作為に倉頡の綴の文字を入れ替えて2,3文字目がなるべく繋がる（同時打鍵できる）入れ替え方を探す。

`source` は倉頡輸入法の定義表の所在。`table/cj5-20000.txt`など。複数指定も可。

`[<keys> <chars>]`の各 `keys`と`chars`の組はその中で無作為に入れ替えられる。
例えば `b x a z dk mh` なら `b`（鍵盤上の`B`） は `x` （倉頡の「難」）、 `a` は `z` に固定され、`d->m k->h` または `d->h k->m` のいずれかとなる。残り物も残り物同士で入れ替わる。

`j` は、ある無作為な入れ替え方について `spell/stat_for_swap` の概算から同時打鍵できない綴数を数える前処理の試行を何回行うか。

`k` は、前処理でたまった `j`個の入れ替え方から最も効率的なものを選び、あとから小小輸入法の定義を書き出せる形で`.auto/???.log`に記録する試行を何回行うか。

最後に `k` 個の上澄みからまた一番効率的な配列を選び、見やすく出力する。

ここまでを無限に繰り返す。

## `cargo run w level <log line>`

`w` は `write` から。
`<log line>`を読んで対応する入れ替え方と倉頡定義文件を用いて `.out/???` に定義表を書き出す。ただし先頭には入れ替え方の文件形式が載る。

`<log line>` には `.auto/???.log` の一行の内容を入れる。

## `cargo run rg level <head_n> <log file>`

`<log file>`は無作為な試行が記録される `.auto/???.log` のこと。

`rg ^[0-9] <log file> | sort -n | head -n <head_n>` で`score`の低い=同時打鍵可能な綴が多い順に並び替えて先頭の`<head_n>`行を取得したのち、
各行の入れ替え方を見やすく表示する。

記録が溜まってきたときに実際の配列を確認してどれを出力するか決めるのに使える。

# 各文件説明
書式は code block に
## `.auto/???.log`
```
<score> <swap> [<source>]
...
```
`score` は同時打鍵不能な正式綴（`^`が前置された、熟語を作るときに参照する用の綴）数

`swap` は入れ替え方

`source` は元にした倉頡定義文件の所在

## `table/...`
加工元の定義文件

`cj5*` が倉頡五代、`xxyx*` が小小音形

## `shuang/xiaoque.txt`
小鶴双拼の定義表。これに下記の声調を足したものが倉頡綴の後ろにくっつく。

- `a` 軽声
- `e` 第四声
- `u` 第三声
- `i` 第一声
- `o` 第二声

[双拼 - Wikipedia](https://ja.wikipedia.org/wiki/%E5%8F%8C%E6%8B%BC)

## `spell/stat_for_swap`
```
<char><char> <int> <char><char> <int> ...
...
```
`table/cj5-20000.txt`における2,3文字目の連続綴の大体の出現数（比）
